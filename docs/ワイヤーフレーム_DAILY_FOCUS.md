# ワイヤーフレーム：DAILY FOCUS

## 1. ワイヤーフレーム作成方法

### 作成ツール

Figmaでワイヤーフレームを作成し、スクリーンショットを撮ってこのページに配置する。  
画像は `./images/wireframe/` フォルダに保存する。

### トップページ ワイヤーフレーム構成案

#### ページの目的

初めて訪問したユーザーに DAILY FOCUS の価値を短時間で伝え、サービス理解から新規登録・利用開始へ自然につなげる。

#### ターゲット

20〜30代の知的労働者や学習者で、やることが多く優先順位付けに悩みやすい人。  
シンプルで使いやすい ToDo・集中支援サービスを探しており、複雑な管理機能よりも「今日やるべきことをすぐ決めて集中できる体験」を求めている。

#### ページに含める要素

- ヘッダー
- ヒーローセクション
- サービス紹介
- 実績
- CTA
- フッター

#### 各セクションの構成提案

| セクション | 配置する要素 | 推奨サイズ | 配置の意図 |
| --- | --- | --- | --- |
| ヘッダー | ロゴ、ナビゲーション、ログイン、新規登録ボタン | 高さ10%程度 | 最初にサービス名と主要導線を提示し、迷わず目的のページへ移動できるようにする |
| ヒーローセクション | キャッチコピー、補足説明、メインCTA、サービスイメージ | 画面全体の30〜35%程度 | ファーストビューで価値を明確に伝え、第一印象で興味を引く |
| サービス紹介 | 課題、解決方法、主要機能3点 | 20〜25%程度 | DAILY FOCUS が何を解決するサービスかを整理して伝える |
| 実績 | 利用イメージ、継続率や満足度の仮数値、レビュー枠 | 10〜15%程度 | 信頼感を補強し、導入への心理的ハードルを下げる |
| CTA | 登録促進メッセージ、無料ではじめるボタン | 10〜15%程度 | 比較検討後の最後の後押しを行い、行動へつなげる |
| フッター | 利用規約、プライバシー、問い合わせ、SNSリンク | 高さ10%程度 | 補足導線と信頼情報を整理して配置する |

### Figmaでの作成ガイド

- フレームサイズ
  - PC版は `1440px` 幅を基準にすると作成しやすい
- カラム構成
  - 12カラムを想定し、左右余白は広めに取る
- セクションの間隔
  - 各セクション間は `80〜120px` 程度の余白を確保する
- ワイヤーフレームの表現
  - 色はグレー基調で統一し、要素の優先順位が伝わる程度に濃淡をつける
- テキスト量
  - 本文はダミーテキストでよく、見出しとCTAだけ実際の文言に寄せる

### セクションごとの具体要素

#### ヘッダー

- 左：ロゴ
- 中央：ナビゲーション3点
  - サービス概要
  - 特長
  - 使い方
- 右：ログイン、新規登録ボタン

#### ヒーローセクション

- 左：キャッチコピー
- 左下：補足説明
- 左下：CTAボタン2つ
  - 無料ではじめる
  - 使い方を見る
- 右：アプリ画面モックまたはダッシュボードのイメージ

#### サービス紹介

- 3カラムで機能紹介カードを並べる
- 各カードに以下を配置する
  - アイコンプレースホルダー
  - 機能名
  - 1〜2行の説明

#### 実績

- 数値カード2〜3枚
  - 継続率
  - 満足度
  - 利用時間など
- 右側または下部にレビュー風のコメント枠を1つ置く

#### CTA

- 左に短い訴求文
- 右に大きめのCTAボタン
- 「今すぐ始める」「無料で試す」など、行動が明確な文言にする

#### フッター

- 利用規約
- プライバシーポリシー
- お問い合わせ
- SNSリンク

### 配置意図の説明

- ヘッダー
  - ページ最上部にロゴ、ナビゲーション、登録ボタンを配置することで、初回訪問者でもサービス理解と登録行動の両方に迷わず進めるようにしている。
- ヒーローセクション
  - ファーストビューにキャッチコピー、説明文、CTAボタン、画面イメージをまとめて配置することで、DAILY FOCUS の価値を短時間で理解できるようにしている。
- サービス紹介
  - ヒーロー直下に「DAILY FOCUSでできること」を配置することで、興味を持ったユーザーがそのまま主要機能を確認できる流れにしている。機能を箇条書きで見せることで、シンプルさと使いやすさが直感的に伝わる。
- 実績
  - サービス紹介の後に「DAILY FOCUSが続けやすい理由」を配置することで、機能理解の次に継続価値を伝える構成にしている。継続率や満足度、集中時間などを並べることで、サービス利用後の効果をイメージしやすくしている。
- CTA
  - ページ終盤に再度 CTA を配置することで、情報を読み終えたユーザーがそのまま登録行動に移りやすいようにしている。最後に強い訴求を置くことで、行動の後押しを行う役割を持たせている。
- フッター
  - フッターにはサービス概要、サポート、規約、お問い合わせを整理して配置し、利用前の不安を減らす構成にしている。必要な情報へすぐアクセスできることで、サイト全体の信頼感向上につながる。

## 3. 下層ページ

### 下層ページ ワイヤーフレーム

【サービス概要ページ】  
<img src="./images/wireframe/page1-pc.png" alt="サービス概要ページ ワイヤーフレーム">  
配置意図：
サービスの目的と解決する課題を整理して伝えるページであるため、上部に概要、その下に課題と解決策を並べる構成にする。情報を順序立てて配置することで、初見のユーザーでも DAILY FOCUS の価値を理解しやすくする。

```text
+----------------------------------------------------------------------------------+
| Header                                                                           |
+----------------------------------------------------------------------------------+
| Page Title: サービス概要                                                         |
| Intro Text                                                                       |
+----------------------------------------------------------------------------------+
| 課題セクション                                                                   |
| [課題1] [課題2] [課題3]                                                          |
+----------------------------------------------------------------------------------+
| 解決策セクション                                                                 |
| [解決策1] [解決策2] [解決策3]                                                    |
+----------------------------------------------------------------------------------+
| DAILY FOCUSの特長                                                                |
| [Feature Card] [Feature Card] [Feature Card]                                     |
+----------------------------------------------------------------------------------+
| CTA                                                                              |
+----------------------------------------------------------------------------------+
| Footer                                                                           |
+----------------------------------------------------------------------------------+
```

【使い方ページ】  
<img src="./images/wireframe/page2-pc.png" alt="使い方ページ ワイヤーフレーム">  
配置意図：
利用開始までのハードルを下げることが目的のため、操作の流れを上から順に追えるレイアウトにする。ステップ形式で並べることで、ユーザーが「自分にも使えそう」と感じやすくなる。

```text
+----------------------------------------------------------------------------------+
| Header                                                                           |
+----------------------------------------------------------------------------------+
| Page Title: 使い方                                                               |
| Intro Text                                                                       |
+----------------------------------------------------------------------------------+
| STEP 1  今日のフォーカスを決める                                                 |
| [Image]                                  [Short Description]                     |
+----------------------------------------------------------------------------------+
| STEP 2  集中セッションを開始する                                                 |
| [Short Description]                      [Image]                                 |
+----------------------------------------------------------------------------------+
| STEP 3  振り返りを記録する                                                       |
| [Image]                                  [Short Description]                     |
+----------------------------------------------------------------------------------+
| FAQ or Small CTA                                                                 |
+----------------------------------------------------------------------------------+
| Footer                                                                           |
+----------------------------------------------------------------------------------+
```

### 下層ページ作成ガイド

- サービス概要ページ
  - 情報整理を重視し、カードや見出しの区切りを明確にする
  - 「課題」から「解決策」へ自然につながる順番で配置する
- 使い方ページ
  - ステップ番号を大きく見せ、視線移動だけで流れが追えるようにする
  - 画像と説明文を交互配置にすると単調さを減らせる
- どちらのページも
  - ヘッダーとフッターはトップページと共通レイアウトにし、サイト全体の統一感を保つ
  - 最後にCTAを置き、登録またはログインへつなげる

### 主要画面の構成案

#### ログイン画面

```text
+--------------------------------------------------------------+
| LOGO                                                         |
+--------------------------------------------------------------+
|                     Login Form Card                          |
|                [email input]                                 |
```

配置意図：
不要な情報を削り、認証行動に集中できる構成にする。中央配置のカード型にすることで入力フォームの視認性を高める。

#### 一覧画面

```text
+----------------------------------------------------------------------------------+
| Side Nav | Header: 今日のタスク                 Search / Filter / Sort           |
+----------------------------------------------------------------------------------+
| Side Nav | [Task Table / Card List]                                              |
|          | [Pagination]                                                          |
```

配置意図：
左にナビゲーション、右に主コンテンツを置く標準的な業務画面構成にし、タスク確認と操作をしやすくする。検索・絞り込み・並び替えは上部固定に近い配置が望ましい。

#### 新規登録画面

```text
+----------------------------------------------------------------------------------+
| Side Nav | Page Title: 新規タスク登録                                            |

| Side Nav | [Title Input]                                                         |
|          | [Description Input]                                                   |
|          | [Date / Priority / Status]                                            |
|          | [Save Button] [Cancel Button]                                         |
+----------------------------------------------------------------------------------+
```

配置意図：
入力フォームは上から順に埋められる構成にし、迷いなく登録できるようにする。保存ボタンは画面下部で目立たせる。

#### 詳細画面

```text
+----------------------------------------------------------------------------------+
| Side Nav | Page Title: タスク詳細                                                |
+----------------------------------------------------------------------------------+
| Side Nav | 基本情報カード                                                        |
|          | メモ / 履歴 / 関連ログ                                                |
|          | [Edit Button]                                                         |
+----------------------------------------------------------------------------------+
```

配置意図：
閲覧を主目的とした画面のため、情報をカードで整理し、更新アクションは右上または下部に限定して配置する。

#### 編集画面

```text
+----------------------------------------------------------------------------------+
| Side Nav | Page Title: タスク編集                                                |
+----------------------------------------------------------------------------------+
| Side Nav | [Editable Form Fields]                                                |
|          | [Update Button] [Delete Button]                                       |
+----------------------------------------------------------------------------------+
```

配置意図：
新規登録画面と似た構造にすることで学習コストを下げる。削除ボタンは更新ボタンと距離を取り、誤操作を防ぐ。

### スマートフォン版作成ガイド

- 横幅は `390px` 前後を想定して作成する
- 1画面内に情報を詰め込みすぎず、カード単位で整理する
- ボタンは親指で押しやすいサイズを確保する
- 下部ナビゲーションを置く場合は、主要機能を3〜5個に絞る
