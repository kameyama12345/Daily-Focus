# ワイヤーフレーム：DAILY FOCUS

## 1. ワイヤーフレーム作成方法

### 作成ツール

Figmaでワイヤーフレームを作成し、スクリーンショットを撮ってこのページに配置する。  
画像は `./images/wireframe/` フォルダに保存する。

### トップページ ワイヤーフレーム構成案

#### ページの目的

初めて訪問したユーザーに DAILY FOCUS の価値を短時間で伝え、サービス理解から新規登録・利用開始へ自然につなげる。

#### ターゲット

20〜30代の知的労働者や学習者で、やることが多く優先順位付けに悩みやすい人。  
シンプルで使いやすい ToDo・集中支援サービスを探しており、複雑な管理機能よりも「今日やるべきことをすぐ決めて集中できる体験」を求めている。

#### ページに含める要素

- ヘッダー
- ヒーローセクション
- サービス紹介
- 実績
- CTA
- フッター

#### 各セクションの構成提案

| セクション | 配置する要素 | 推奨サイズ | 配置の意図 |
| --- | --- | --- | --- |
| ヘッダー | ロゴ、ナビゲーション、ログイン、新規登録ボタン | 高さ10%程度 | 最初にサービス名と主要導線を提示し、迷わず目的のページへ移動できるようにする |
| ヒーローセクション | キャッチコピー、補足説明、メインCTA、サービスイメージ | 画面全体の30〜35%程度 | ファーストビューで価値を明確に伝え、第一印象で興味を引く |
| サービス紹介 | 課題、解決方法、主要機能3点 | 20〜25%程度 | DAILY FOCUS が何を解決するサービスかを整理して伝える |
| 実績 | 利用イメージ、継続率や満足度の仮数値、レビュー枠 | 10〜15%程度 | 信頼感を補強し、導入への心理的ハードルを下げる |
| CTA | 登録促進メッセージ、無料ではじめるボタン | 10〜15%程度 | 比較検討後の最後の後押しを行い、行動へつなげる |
| フッター | 利用規約、プライバシー、問い合わせ、SNSリンク | 高さ10%程度 | 補足導線と信頼情報を整理して配置する |

### Figmaでの作成ガイド

- フレームサイズ
  - PC版は `1440px` 幅を基準にすると作成しやすい
- カラム構成
  - 12カラムを想定し、左右余白は広めに取る
- セクションの間隔
  - 各セクション間は `80〜120px` 程度の余白を確保する
- ワイヤーフレームの表現
  - 色はグレー基調で統一し、要素の優先順位が伝わる程度に濃淡をつける
- テキスト量
  - 本文はダミーテキストでよく、見出しとCTAだけ実際の文言に寄せる

### セクションごとの具体要素

#### ヘッダー

- 左：ロゴ
- 中央：ナビゲーション3点
  - サービス概要
  - 特長
  - 使い方
- 右：ログイン、新規登録ボタン

#### ヒーローセクション

- 左：キャッチコピー
- 左下：補足説明
- 左下：CTAボタン2つ
  - 無料ではじめる
  - 使い方を見る
- 右：アプリ画面モックまたはダッシュボードのイメージ

#### サービス紹介

- 3カラムで機能紹介カードを並べる
- 各カードに以下を配置する
  - アイコンプレースホルダー
  - 機能名
  - 1〜2行の説明

#### 実績

- 数値カード2〜3枚
  - 継続率
  - 満足度
  - 利用時間など
- 右側または下部にレビュー風のコメント枠を1つ置く

#### CTA

- 左に短い訴求文
- 右に大きめのCTAボタン
- 「今すぐ始める」「無料で試す」など、行動が明確な文言にする

#### フッター

- 利用規約
- プライバシーポリシー
- お問い合わせ
- SNSリンク

### 配置意図の説明

- ヘッダー
  - ページ最上部にロゴ、ナビゲーション、登録ボタンを配置することで、初回訪問者でもサービス理解と登録行動の両方に迷わず進めるようにしている。
- ヒーローセクション
  - ファーストビューにキャッチコピー、説明文、CTAボタン、画面イメージをまとめて配置することで、DAILY FOCUS の価値を短時間で理解できるようにしている。
- サービス紹介
  - ヒーロー直下に「DAILY FOCUSでできること」を配置することで、興味を持ったユーザーがそのまま主要機能を確認できる流れにしている。機能を箇条書きで見せることで、シンプルさと使いやすさが直感的に伝わる。
- 実績
  - サービス紹介の後に「DAILY FOCUSが続けやすい理由」を配置することで、機能理解の次に継続価値を伝える構成にしている。継続率や満足度、集中時間などを並べることで、サービス利用後の効果をイメージしやすくしている。
- CTA
  - ページ終盤に再度 CTA を配置することで、情報を読み終えたユーザーがそのまま登録行動に移りやすいようにしている。最後に強い訴求を置くことで、行動の後押しを行う役割を持たせている。
- フッター
  - フッターにはサービス概要、サポート、規約、お問い合わせを整理して配置し、利用前の不安を減らす構成にしている。必要な情報へすぐアクセスできることで、サイト全体の信頼感向上につながる。

## 3. 下層ページ

### 下層ページ ワイヤーフレーム

【機能詳細ページ】  
配置意図：
トップページの「詳しく見る」から遷移する下層ページとして、DAILY FOCUS の主要機能を詳しく伝える構成にしている。上部にページタイトルを置き、その下に「今日のフォーカス」「集中セッション」「振り返り」を順に並べることで、機能の内容と価値を段階的に理解しやすくしている。

```text
+----------------------------------------------------------------------------------+
| Header                                                                           |
+----------------------------------------------------------------------------------+
| Page Title: DAILY FOCUSでできること                                              |
| Intro Text                                                                       |
+----------------------------------------------------------------------------------+
| 今日のフォーカス                                                                 |
| [Image]                                  [Feature Description]                   |
+----------------------------------------------------------------------------------+
| 集中セッション                                                                   |
| [Feature Description]                    [Image]                                 |
+----------------------------------------------------------------------------------+
| 振り返り                                                                         |
| [Image]                                  [Feature Description]                   |
+----------------------------------------------------------------------------------+
| CTA                                                                              |
+----------------------------------------------------------------------------------+
| Footer                                                                           |
+----------------------------------------------------------------------------------+
```

### 下層ページ作成ガイド

- 機能詳細ページ
  - 上部にページタイトルを置き、トップページから遷移したユーザーが何の説明ページかすぐ理解できるようにする
  - 「今日のフォーカス」「集中セッション」「振り返り」を順に並べ、主要機能を段階的に理解できる構成にする
  - 画像と説明文を左右交互に配置することで、単調さを抑えながら読み進めやすくする
  - ヘッダーとフッターはトップページと共通レイアウトにし、サイト全体の統一感を保つ
  - 最後にCTAを置き、詳細確認後に登録またはログインへつなげる

### 主要画面の構成案

#### ログイン画面

```text
+--------------------------------------------------------------+
| DAILY FOCUS                                                  |
+--------------------------------------------------------------+
|                     Login Form Card                          |
|                [メールアドレス入力欄]                         |
|                [パスワード入力欄]                             |
|                [ログインボタン]                               |
|                [補足テキスト]                                 |
|                [外部ログインボタン]                           |
+--------------------------------------------------------------+
```

配置意図：
画面中央にログインカードを配置することで、ユーザーが迷わず認証行動に集中できるようにしている。入力欄、ログインボタン、補足情報を縦に並べることで、視線の流れに沿って自然に操作できる構成にしている。

#### 一覧画面

```text
+----------------------------------------------------------------------------------+
| Header: スケジュール / おすすめ / テンプレート / 新規タスク                       |
+----------------------------------------------------------------------------------+
| Time Column | 今日のスケジュール一覧                                              |
|             | [タスクカード] [タスクカード] [タスクカード]                        |
+----------------------------------------------------------------------------------+
```

配置意図：
時間軸に沿ってタスクを一覧表示することで、ユーザーが1日の流れを直感的に把握できる構成にしている。上部にはおすすめスケジュールやテンプレート、新規タスク作成ボタンを配置し、確認と追加の両方を行いやすくしている。

#### 新規登録画面

```text
+----------------------------------------------------------------------------------+
| DAILY FOCUS                                                                      |
+----------------------------------------------------------------------------------+
|                     Sign Up Form Card                                            |
|                [メールアドレス入力欄]                                             |
|                [パスワード入力欄]                                                 |
|                [新規登録ボタン]                                                   |
|                [利用規約・補足テキスト]                                           |
|                [Google / Appleログインボタン]                                     |
+----------------------------------------------------------------------------------+
```

配置意図：
新規登録画面もログイン画面と同様に中央カード型にすることで、初回利用者でも迷わず入力を進められるようにしている。入力項目を最小限に絞り、登録ボタンを目立たせることで、利用開始までのハードルを下げる構成にしている。

#### 登録画面について

今回のワイヤーフレームでは、課題要件にある「登録画面」を会員登録画面として作成している。DAILY FOCUS では利用開始前にアカウント作成が必要であるため、ユーザー登録の導線を優先して画面化した。

#### 詳細画面

```text
+----------------------------------------------------------------------------------+
| Task Detail Card                                                                 |
+----------------------------------------------------------------------------------+
| タスク名                                                                         |
| 時間                                                                            |
| カテゴリ                                                                         |
| 進捗 / ステータス                                                                |
+----------------------------------------------------------------------------------+
```

配置意図：
一覧画面から選択したタスクの内容をひと目で確認できるように、必要な情報だけをコンパクトにまとめた構成にしている。詳細を独立して見せることで、予定確認や内容把握をすばやく行える。

#### 編集画面

```text
+----------------------------------------------------------------------------------+
| Page Title: タスク編集                                                           |
+----------------------------------------------------------------------------------+
| [タイトル入力欄]                                                                  |
| [開始時間] [終了時間]                                                             |
| [カテゴリ] [優先度]                                                               |
| [メモ入力欄]                                                                      |
| [完了チェック] [削除] [キャンセル] [保存]                                         |
+----------------------------------------------------------------------------------+
```

配置意図：
タスクの編集に必要な項目を上から順に並べることで、ユーザーが迷わず修正できるようにしている。保存、削除、キャンセルを画面下部にまとめることで、編集後の行動を分かりやすくしている。
