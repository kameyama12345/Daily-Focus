# DAILY FOCUS 改善メモ（NG中心）

作成日: 2026-04-21  
目的: 評価シートのうち、まず対応すべき `NG` 項目を中心に整理し、次に見る `要確認` を補足として残す。

---

## 1. 今回の方針

今回の改善では、`要確認` を広く拾うよりも、**まず NG が付いている項目を優先して直す**。  
そのため、このメモでは以下の順で整理する。

1. NG項目を明確にする  
2. それぞれの問題点を言語化する  
3. 直し方と完了条件を決める  
4. 要確認項目は補足として残す  

---

## 2. NG項目

画像の評価表から見て、まず直す対象として強く意識すべき項目は以下。

1. フォーム入力補助・エラー表示  
2. 静的資産の整理  
3. テンプレート命名と構造  
4. 本番不要ファイルの混在  

---

## 3. NG項目ごとの整理

### NG-1. フォーム入力補助・エラー表示

**評価メモ**
- `login / signup は見た目先行で、実装状況が分かりにくい`

**現状の問題**
- ログイン画面・会員登録画面は見た目は整っている
- ただし、未入力時やエラー時にどうなるかが画面上で分からない
- 「フォームとしてどこまで実装しているか」を説明しにくい

**対応方針**
- login / signup に入力補助テキストを入れる
- 未入力エラーの表示例を入れる
- 必須項目が分かるようにする
- 実装モックでも「どう使う画面か」が伝わる状態にする

**完了条件**
- 未入力時のエラー見本がある
- 入力欄ごとの役割が分かる
- 提出時に「入力補助・エラー表示を考慮している」と説明できる

---

### NG-2. 静的資産の整理

**評価メモ**
- `ポートフォリオ repo としては何が現行成果物か迷いやすい`

**現状の問題**
- 制作ドキュメント、Next 本体、提出用静的画面、ビルド成果物などが混在している
- 初見の人が「どこを見ればいいか」をすぐ判断しにくい

**対応方針**
- 提出対象を README で明示する
- `_制作ドキュメント` と `_フロントエンド実装` を中心に見せる
- 補助的なフォルダは役割を明記するか、不要なら削る

**完了条件**
- README を見れば提出対象がすぐ分かる
- 「完成版」「提出用静的画面」「制作ドキュメント」の区別が明確
- repo を開いた人が迷いにくい

---

### NG-3. テンプレート命名と構造

**評価メモ**
- `見せる対象を絞ると repo の印象がかなり良くなる`

**現状の問題**
- 開発途中の名前や補助フォルダが混じり、提出物としての見通しが少し弱い
- 「本番」「提出用」「試作」の境界が伝わりにくい部分がある

**対応方針**
- 提出対象の命名を統一する
- 入口となる README と `_フロントエンド実装/index.html` で役割を整理する
- どの画面が提出物で、どの画面が完成版かを明記する

**完了条件**
- ファイル名とフォルダ名だけで役割が想像しやすい
- `_制作ドキュメント` と `_フロントエンド実装` が提出物として見える
- 初見の人が構成を追いやすい

---

### NG-4. 本番不要ファイルの混在

**評価メモ**
- `ポートフォリオ用の最終提出repoとしては整理したい`

**現状の問題**
- `logs`、`out`、`auth-demo`、`extension` など、提出時に不要なものが混在している
- 実装成果と関係の薄いものが repo のノイズになる

**対応方針**
- 不要ファイルを削除するか、少なくとも提出対象外として整理する
- 最終的に残すものと残さないものを判断する
- 提出時に見せるべき範囲をしぼる

**完了条件**
- repo を開いたときに不要物が目立たない
- 提出対象外ファイルの扱いを説明できる
- ポートフォリオとしての見た目がすっきりする

---

## 4. NG対応の作業順

### 先にやること

- [ ] login / signup にエラー表示と補助表示を入れる
- [ ] README で提出対象を明確にする
- [ ] `_フロントエンド実装/index.html` の案内を最終整理する
- [ ] 不要ファイルを残すか削るか判断する

### 最後に確認すること

- [ ] 提出先が README を見て迷わないか
- [ ] 認証画面が「見た目だけ」でなく入力補助もある状態か
- [ ] リポジトリが成果物中心に見えるか

---

## 5. 要確認項目（補足）

NGの次に見ておく項目は以下。

- スマホ操作性  
  - 見た目は対応しているが、実機での押しやすさは再確認したい
- 基本アクセシビリティ  
  - ラベル、フォーカス、キーボード操作の確認を追加したい
- 公開URLの安定表示  
  - GitHub Pages の主要ページが安定して見られるか再確認したい

---

## 6. 今回のゴール

今回の改善のゴールは、`要確認を広く拾うこと` ではなく、  
**NG項目を直して、提出物として説明しやすい状態にすること** である。

そのため、まずは以下を達成する。

- 認証画面の入力補助・エラー表示を補強する
- README と提出フォルダの見せ方を整理する
- 不要ファイルの扱いを決める
- リポジトリ全体を「提出物」として見やすくする
