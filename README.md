# Daily Focus

DAILY FOCUS は、1日の予定と集中タスクを見える化するデイリープランナーの制作課題です。  
このリポジトリは、課題提出用に整理した `submission` ブランチの内容を想定しています。  
提出物と完成版アプリ実装を中心に残し、最終提出で不要な試作フォルダやビルド生成物は repo の見通しを優先して除外しています。

## ルート構成

提出時に主に見る場所は次のとおりです。

- `_制作ドキュメント`: 企画・調査・設計・振り返りの提出資料
- `_フロントエンド実装`: 提出用に整理した静的画面
- `src`: 完成版アプリ実装
- `README.md`: 提出物の見方と区分の説明

## 提出時の見方

提出確認は次の順で見ると分かりやすい構成です。

1. `./README.md`
2. `./_制作ドキュメント`
3. `./_フロントエンド実装`
4. `./src/app` と `./src/components`

## 最初に見る場所

提出物として最初に確認してほしいのは次の 2 つです。

1. `./_制作ドキュメント`
2. `./_フロントエンド実装`

この 2 フォルダが、企画から画面設計、提出用画面までを確認するための中心です。

## 区分の考え方

- 提出物: 課題提出として見せるために整理した資料と静的画面
- 完成版: Next.js で構成した現行のアプリ実装
- 除外対象: 試作フォルダ、ログ、ビルド生成物など提出時に不要なもの

## 提出対象の整理

### 1. 制作ドキュメント

フォルダ: `./_制作ドキュメント`

企画、調査、設計、ワイヤーフレーム、デザイン方針、振り返りをまとめた提出用ドキュメントです。

- `01_企画提案書_DAILY_FOCUS.md`
- `02_マーケットリサーチ_DAILY_FOCUS.md`
- `03_ペルソナシート_DAILY_FOCUS.md`
- `04_サイトマップ_DAILY_FOCUS.md`
- `05_ワイヤーフレーム_DAILY_FOCUS.md`
- `06_デザインガイドライン_DAILY_FOCUS.md`
- `07_振り返り・技術記事_DAILY_FOCUS.md`

### 2. 提出用静的画面

フォルダ: `./_フロントエンド実装`

画面構成を提出用に確認しやすくした HTML / CSS の静的画面です。  
「どんな画面を作ったか」を一覧しやすい成果物として扱っています。

- `01_提出用画面一覧.html`
- `02_ログイン画面.html`
- `03_会員登録画面.html`
- `04_ダッシュボード画面.html`
- `05_タスク詳細画面.html`
- `06_タスク編集画面.html`
- `提出用共通スタイル.css`
- `提出用補助スクリプト.js`

### 3. 完成版アプリ実装

フォルダ: `./src/app`, `./src/components`, `./src/hooks`, `./src/lib`

Next.js で構成した現行のアプリ実装です。  
課題提出の補足として、実装版を確認したい場合はこちらを参照してください。  
提出物を目立たせるため、実装本体は `src` 配下に集約しています。

### 4. 提出対象外として除外したもの

最終提出 repo では、次のようなフォルダは残さない方針にしています。

- `out`
- `logs`
- `auth-demo`
- `extension`

これらはビルド生成物、ローカルログ、試作モック、補助実験用ファイルであり、提出物の中心ではありません。

また、内部作業用メモや一時検証用ファイルも `submission` では残さない方針にしています。

## フォルダごとの役割

| フォルダ / ファイル | 区分 | 役割 |
| --- | --- | --- |
| `README.md` | 提出物補助 | 提出物の見方と repo 全体の説明 |
| `_制作ドキュメント` | 提出物 | 企画・設計・調査・振り返りの提出資料 |
| `_フロントエンド実装` | 提出物 | 提出用の静的画面 |
| `src/app` / `src/components` / `src/hooks` / `src/lib` | 完成版 | Next.js の現行アプリ実装 |
| `.github/workflows` | 補助 | `main` ブランチ公開用の GitHub Pages 設定 |
| `out` | 除外対象 | GitHub Pages 用のビルド成果物 |
| `auth-demo` | 除外対象 | 認証画面の試作モック |
| `extension` | 除外対象 | ブラウザ拡張の試作ファイル |
| `logs` | 除外対象 | ローカル実行時のログ |
| `.next` | 補助 | Next.js のローカルビルドキャッシュ |

## 迷ったらここを見る

- 課題の提出内容を確認したい: `./_制作ドキュメント`
- 提出用の画面一覧を確認したい: `./_フロントエンド実装`
- 完成版の実装コードを確認したい: `./src/app` と `./src/components`
- 提出対象外の扱いを確認したい: この README の「提出対象外として除外したもの」
