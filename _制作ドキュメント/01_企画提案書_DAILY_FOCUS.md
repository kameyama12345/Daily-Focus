# 企画提案書：DAILY FOCUS

## 1. プロジェクト概要

### サービス名
- **DAILY FOCUS**

### コンセプト
- **キャッチコピー**：**「今日やることを、ひとつに絞る。」**
- **説明（1〜2文）**：DAILY FOCUS は、毎日「最重要の1つ」を決め、集中セッションで実行し、短い振り返りまでを一気通貫で支援するミニマルな生産性サービスです。ToDo の肥大化や意思決定疲れを減らし、行動の継続と達成感を積み上げます。

### ターゲットユーザー
- リモートワーク/ハイブリッドで働く知的労働者（タスクが多く、集中が分断されがちな層）
- 学生・資格学習者（学習計画はあるが、日々の実行が続かない層）
- 小規模チームの個人（複雑なPMツールは不要だが、毎日の優先順位付けが必要な層）

## 2. 背景と課題

### 市場の現状（市場の現状と課題）
- 生産性/習慣化/タイマー/ToDo アプリは多数存在し、選択肢が過多になっています。
- 高機能化により「設定・整理・入力」に時間がかかり、継続率が下がりやすい傾向があります。
- リモートワークの普及で、会議・チャット通知・マルチタスクによる**コンテキストスイッチ**が増え、深い集中（Deep Work）が阻害されがちです。
- 既存ツールは「管理（整理）」と「実行（集中）」が分断されやすく、行動に結びつきにくい課題があります。

### 解決すべき課題（本プロジェクトで解決を目指す具体的な課題）
1. **優先順位が決められず、重要タスクに着手できない**（毎日の意思決定コストが高い）
2. **集中の開始ハードルが高く、細切れ作業になって成果が出にくい**（時間の使い方が分散する）
3. **振り返りが続かず、改善が蓄積しない**（再現性のある習慣にならない）

## 3. 提案するソリューション

### サービスの特長（競合と差別化する特長）
1. **「今日の最重要1つ」ファースト設計**
   - タスクを増やすのではなく、毎朝（または開始時）に「今日のフォーカス」を1つ決める体験を中心に設計。
2. **決める→集中する→振り返る を1画面導線で完結**
   - ToDo管理とタイマー、実行ログ、短いレビューを繋げ、行動までの摩擦を最小化。
3. **ミニマル運用で続く、データに基づく自己理解**
   - 入力項目を絞りつつ、集中時間・達成率・時間帯などを可視化して「続く理由」を作る。

## 4. 制作範囲

### Webサイト（フロントエンド）の範囲（制作するページ一覧と概要）
- **LP（ランディングページ）**
  - 価値提案、機能紹介、料金（将来）、FAQ、問い合わせ導線
- **サインアップ / ログイン**
  - メール/ソーシャルログイン（候補）、パスワード再設定
- **ダッシュボード（Today）**
  - 今日のフォーカス設定、今日の予定/タスク表示、開始ボタン
- **フォーカスセッション**
  - タイマー（例：25/50分）、休憩、セッションメモ、完了報告
- **タスク（Backlog）**
  - タスク作成/整理、重要度、期限（任意）、アーカイブ
- **インサイト**
  - 集中時間推移、達成率、時間帯別の傾向、連続記録（streak）
- **設定**
  - タイマー設定、通知、表示、エクスポート、アカウント
- **ヘルプ/利用規約/プライバシーポリシー**

### Webアプリケーション（サーバーサイド）の範囲（機能概要）
- **認証・ユーザー管理**
  - ユーザー登録/ログイン、プロフィール、退会
- **データ管理**
  - 今日のフォーカス、タスク、セッションログ、振り返りログのCRUD
- **分析/集計**
  - 期間別集計（週/月）、達成率、集中時間、時間帯別傾向
- **通知（任意）**
  - リマインド（開始/休憩/終了、日次リセット）
- **エクスポート（任意）**
  - CSV/JSON でのログ出力

## 5. 使用技術

### フロントエンド
- Next.js（App Router）/ React
- TypeScript
- Tailwind CSS
- アイコン：lucide-react
- 日付処理：date-fns

### バックエンド（候補）
- Next.js Route Handlers / Server Actions
- DB：PostgreSQL（例：Vercel Postgres / Supabase など）
- ORM：Prisma（導入する場合）
- 認証：Auth.js（導入する場合）

### インフラ/運用（候補）
- デプロイ：Vercel
- 監視/ログ：Vercel Logs + 追加ツール（必要に応じて）
